# Deploying the Next.js Frontend (Vercel)

## One-time setup

1. Push this repo to GitHub (or ensure it is already there).
2. Go to [vercel.com](https://vercel.com) → **New Project** → **Import** the repository.
3. Under **Configure Project**, set:
   - **Root Directory:** `web`
   - **Framework Preset:** Next.js (auto-detected)
   - **Build Command:** `npm run build` (auto-filled)
   - **Output Directory:** `out` (auto-filled from `vercel.json`)
4. **No environment variables** are needed — the app reads static JSON from `web/public/data/`.
5. Click **Deploy**. Vercel shows the live URL when done.
6. Copy the URL and update the Frontend badge in `README.md`.

## Refreshing data

The frontend reads pre-exported JSON. To update it with the latest sample bundle:

```powershell
python scripts\build_web_export.py
```

Commit the updated `web/public/data/*.json` files and push — Vercel auto-redeploys.

## Local development

```powershell
cd web
npm install
npm run dev
# → http://localhost:3000
```

## Local static build (mirrors Vercel)

```powershell
cd web
npm run build
# output in web/out/
```

## Parity test (TS ↔ Python ranking)

```powershell
cd web
npm test
```

Asserts that `rankMandis()` in TypeScript matches `score_recommendations()` in Python
within 0.01 INR/qtl at the default farmer location (Nashik 19.9975, 73.7898).
Run after any change to `src/lib/transport.ts` or `configs/recommendation.yaml`.

## Architecture

```
web/public/data/*.json   ← static, committed, generated by build_web_export.py
web/src/app/             ← Next.js App Router pages (coverage, forecast, recommend)
web/src/components/      ← React components
web/src/lib/transport.ts ← port of src/mandipulse/recommend/engine.py
web/src/lib/data.ts      ← JSON loaders
```

The transport-cost slider re-ranks mandis client-side in TypeScript on every change
(no server call). The ranking logic is a direct port of `engine.py` — verified by the
parity test.
