# MandiPulse Implementation Plan

## Current State

The Onion/Maharashtra 7-day decision loop is implemented and portfolio-ready:

- Cached CEDA data can be transformed into clean panel and feature-table artifacts.
- Temporal baselines, LightGBM, residual-LightGBM, and uncertainty reports are implemented.
- Moving-average remains the shipped forecaster because it wins on held-out test MAE.
- Transport-aware recommendations and regret@K backtesting are implemented.
- Streamlit, FastAPI, and static Next.js surfaces read the same demo data contract.
- CI now runs Python lint/format/tests and web parity/build checks.

## Artifact Policy

Keep these committed:

- `data/sample/`: slim demo bundle for clone-runnable Streamlit/API behavior.
- `web/public/data/*.json`: static frontend data generated by `scripts/build_web_export.py`.
- Markdown reports under `reports/`.

Keep these local and ignored:

- Raw/processed data under `data/raw/`, `data/interim/`, and `data/processed/`.
- Full generated CSVs under `artifacts/`.
- MLflow runs under `mlruns/`.

## Rules While Implementing

- Keep the project framed as mandi decision intelligence, not generic crop-price prediction.
- Keep temporal validation and leakage tests intact.
- Do not require secrets for public demos.
- Update README, tracker, API spec, and architecture docs when public contracts change.
